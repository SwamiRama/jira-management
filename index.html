üîç Detailed Explanations

What are "Schemes"?

Schemes are like templates or blueprints. Instead of configuring everything from scratch for each project, you create a scheme once and apply it to multiple projects. Think of them as reusable configuration packages.

When do you need what?

  • New Project: Work Type Scheme, Workflow Scheme, Screen Scheme
  • New Fields: Custom Fields ‚Üí Field Configuration ‚Üí Field Configuration Scheme
  • Different Permissions: Permission Scheme
  • Different Notifications: Notification Scheme
  • Customer Portal: Forms and Data Connections

Typical Setup Order:

1. Create Foundation Components (Work Types, Workflows, Fields)
2. Create Schemes (bundle the foundation components)
3. Apply Schemes to Projects (assign to your service projects)

Real-World Example:

IT Support Team Setup:
‚Ä¢ Work Types: Incident, Service Request, Problem, Change
‚Ä¢ Workflow: New ‚Üí Assigned ‚Üí In Progress ‚Üí Resolved ‚Üí Closed
‚Ä¢ Custom Fields: Priority, Category, Affected Users, Resolution
‚Ä¢ Screens: Create Screen (basic info), Edit Screen (all fields), View Screen (read-only)

The Relationship Chain:

Foundation ‚Üí Schemes ‚Üí Projects
Everything flows from basic components into reusable schemes, which are then applied to projects. This allows you to maintain consistency while having flexibility for different project needs.
